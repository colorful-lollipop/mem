背压机制说明

场景触发条件处理机制恢复条件
请求队列满 SubmitWorker TryPushRequest返回QueueFull WaitForRequestCredit() poll(reqCreditEventFd)阻塞 Dispatcher消费请求后signal
响应队列满 ResponseWriter PushResponse返回QueueFull WaitForResponseCredit() poll(respCreditEventFd)阻塞 ResponseWorker消费响应后signal
背压机制价值
  • 防止内存溢出: 队列满时阻塞写入，避免无限增长
  • 自动流控: 生产者和消费者速度自动匹配
  • 零CPU空转: 基于EventFd的阻塞等待，无请求时CPU占用为0
  • 优先级保护: 高优请求使用独立的Ring和EventFd，不受普通请求影响
