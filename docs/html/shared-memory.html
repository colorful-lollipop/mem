共享内存结构

共享内存是客户端和服务端通信的核心载体，采用无锁环形队列设计。

Ring Entry 结构

RequestRingEntry (8192 bytes)

struct RequestRingEntry {
    // Header: 24 bytes
    uint64_t requestId;        // 请求唯一ID
    uint32_t execTimeoutMs;    // 执行超时时间
    uint16_t opcode;           // 操作码 (ScanFile=102)
    uint8_t  priority;         // 优先级 (0=Normal, 1=High)
    uint8_t  reserved0;
    uint32_t payloadSize;      // 有效载荷大小
    
    // Payload: 8168 bytes
    std::array<uint8_t, 8168> payload;
};

ResponseRingEntry (8192 bytes)

struct ResponseRingEntry {
    // Header: 36 bytes
    uint64_t requestId;        // 对应请求ID
    uint32_t statusCode;       // 状态码
    uint32_t eventDomain;      // 事件域
    uint32_t eventType;        // 事件类型
    uint32_t flags;
    uint32_t resultSize;
    ResponseMessageKind messageKind;  // Reply(0) / Event(1)
    
    // Payload: 8156 bytes
    std::array<uint8_t, 8156> payload;
};

BootstrapHandles (引导句柄)

客户端和服务端建立连接时通过 Unix Socket 传递的句柄集合：

字段说明方向
shmFd共享内存文件描述符Client <-> Server
highReqEventFd高优先级请求事件通知Client -> Server
normalReqEventFd普通优先级请求事件通知Client -> Server
respEventFd响应/事件通知Server -> Client
reqCreditEventFd请求队列空位通知Server -> Client
respCreditEventFd响应队列空位通知Client -> Server

EventFd 详细作用

EventFd触发场景监听方作用价值
highReqEventFdClient写入High Ring后signalServer DispatcherThread减少CPU空转，Server等待通知处理高优请求
normalReqEventFdClient写入Normal Ring后signalServer DispatcherThread优先级隔离，高优不受普通请求影响
respEventFdServer写入Response Ring后signalClient ResponseWorker异步响应，Client收到通知立即读取
reqCreditEventFdServer消费请求后signalClient SubmitWorker背压控制，队列满时Client阻塞等待
respCreditEventFdClient消费响应后signalServer ResponseWriter流控平衡，响应队列满时Server阻塞

为什么使用5个EventFd而不是轮询？
  1. 性能: 轮询需要CPU不断检查共享内存状态，EventFd基于内核通知，无请求时CPU占用为0
  2. 实时性: 事件通知是即时的，轮询有检测延迟
  3. 背压: EventFd天然支持阻塞等待，简化流控实现
  4. 优先级: 独立的High/Normal EventFd实现真正的优先级隔离
