Virus Executor Service 代码串讲文档

本文档详细梳理了基于共享内存RPC的病毒扫描服务架构，以 scanFile 为例讲解核心流程。

文档导航

本文档采用分层架构视角，从应用层到基础设施层逐层剖析：

  • 应用层 - VesClient API 和业务接口
  • 框架层 - RpcClient/RpcServer 生命周期管理
  • 传输层 - 共享内存 Session 管理
  • 基础设施层 - EventFd、进程管理

核心设计要点
  • 双优先级队列：独立的 High/Normal Request Ring，避免普通请求阻塞高优先级请求
  • EventFd 通知：基于 Linux eventfd 的轻量级进程间通知，替代轮询
  • Robust Mutex：使用 PTHREAD_MUTEX_ROBUST 处理客户端崩溃后的锁恢复
  • 状态机驱动：ClientRecoveryState 管理 7 种生命周期状态
  • 自动恢复：支持 3 种恢复策略（onFailure/onIdle/onEngineDeath）
