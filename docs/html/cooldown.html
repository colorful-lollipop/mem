客户端生命周期状态机

本章定位

本章详细讲解 RpcClient 的生命周期状态管理，涵盖 7 种状态和完整的转换规则。

状态转换说明

初始化阶段

① Uninitialized → Recovering

触发条件：调用 Init() 初始化客户端

系统行为：启动后台线程（SubmitWorker、ResponseWorker、Watchdog），开始尝试建立会话

用户感知：初始化是异步的，Init() 会立即返回，后台自动进入恢复流程

正常恢复阶段

② Recovering → Active

触发条件：OpenSession() 成功建立连接

系统行为：完成共享内存映射，会话就绪，可以正常收发请求

用户感知：首个请求可能需要等待恢复完成，后续请求正常处理

③ Recovering → NoSession

触发条件：OpenSession() 失败（如 SA 启动失败）

系统行为：放弃恢复，标记为无可用会话

用户感知：新请求会立即返回 PeerDisconnected 错误，不会自动重试

异常处理阶段（从 Active 出发）

④ Active → Cooldown（延迟恢复）

触发条件：发生异常（引擎崩溃或请求超时），且策略配置为 Restart(delayMs>0)

典型场景：默认配置下引擎崩溃后进入 200ms 冷却期

系统行为：

  • 关闭当前会话，标记所有待处理请求为失败
  • 不立即尝试重连，而是等待指定的冷却时间
  • 冷却期间新请求会阻塞等待，不会触发新的恢复尝试

设计目的：防止故障服务被频繁拉起，给系统留出稳定时间

⑤ Active → Recovering（立即恢复）

触发条件：发生异常，且策略配置为 Restart(0)

系统行为：关闭当前会话后立即开始尝试建立新会话，无延迟

用户感知：新请求会阻塞直到新会话建立完成

⑥ Active → NoSession（不恢复）

触发条件：发生异常，但策略配置为 Ignore 或未配置恢复策略

系统行为：关闭会话后不再尝试恢复

用户感知：后续请求快速失败，需要外部干预才能恢复

⑦ Active → IdleClosed（空闲关闭）

触发条件：长时间无请求（默认 60 秒），且策略配置为 IdleClose

系统行为：主动关闭会话以节省资源

用户感知：无感知；新请求到达时会自动触发恢复（见转换⑩）

恢复触发阶段

⑧ Cooldown → Recovering（冷却结束）

触发条件：冷却时间到达，或有新请求进来

系统行为：开始尝试建立新会话

关键特性：冷却期间的新请求不会触发多次恢复尝试，而是统一等待当前冷却完成

⑨ NoSession → Recovering（外部触发）

触发条件：外部调用 RequestExternalRecovery()

系统行为：重新开始恢复流程

典型场景：运维人员手动触发或定时任务检测到服务恢复后重试

⑩ IdleClosed → Recovering（新请求触发）

触发条件：新请求到达

系统行为：自动开始恢复流程，对用户透明

与 NoSession 的区别：IdleClosed 可以被新请求自动触发恢复，而 NoSession 必须外部触发

关闭阶段

⑪ 任意状态 → Closed（关闭）

触发条件：调用 Shutdown()

系统行为：

  • 关闭当前会话
  • 停止所有后台线程
  • 清理所有待处理请求

重要特性：Closed 是终态，一旦进入无法再转移到其他状态

默认策略配置
MemRpc::RecoveryPolicy BuildRecoveryPolicy() {
    // 执行超时后：延迟200ms恢复
    policy.onFailure = [](const RpcFailure& failure) {
        if (failure.status == StatusCode::ExecTimeout) {
            return RecoveryDecision{RecoveryAction::Restart, 200};
        }
        return RecoveryDecision{RecoveryAction::Ignore, 0};
    };
    
    // 引擎死亡后：延迟200ms恢复
    policy.onEngineDeath = [](const EngineDeathReport& report) {
        return RecoveryDecision{RecoveryAction::Restart, 200};
    };
    
    // 空闲60秒后：关闭会话
    policy.onIdle = [](uint64_t idleMs) {
        if (idleMs >= 60000) {
            return RecoveryDecision{RecoveryAction::IdleClose, 0};
        }
        return RecoveryDecision{RecoveryAction::Ignore, 0};
    };
}
