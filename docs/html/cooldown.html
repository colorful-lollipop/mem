冷却(Cooldown)机制

本章定位

本章详细讲解冷却机制的原理和状态机，是第4.1.1节冷却流程的深入展开。

冷却机制用于在故障恢复后避免立即重试导致的频繁抖动，给系统留出稳定时间。

冷却状态机

stateDiagram-v2 [*] --> Uninitialized Uninitialized --> Recovering : Init() Recovering --> Active : 恢复成功 Active --> Cooldown : 故障/重启 Cooldown --> Recovering : 冷却结束 Recovering --> NoSession : 恢复失败 NoSession --> [*] Active --> IdleClosed : 空闲超时 IdleClosed --> [*] Active --> Closed : Shutdown() Closed --> [*]

状态说明

状态含义新请求处理
Active会话正常正常提交
Cooldown恢复冷却中（默认200ms）阻塞等待冷却结束
Recovering正在建立新会话阻塞等待恢复完成
NoSession恢复失败，无可用会话快速返回错误

新任务在冷却期的处理逻辑

sequenceDiagram participant App as App participant RC as RpcClient participant CRS as ClientRecoveryState participant ST as SessionTransport App->>RC: ScanFile() RC->>RC: InvokeAsync() RC->>CRS: EnsureLiveSession() CRS->>CRS: 检查状态:Cooldown CRS-->>RC: 返回CooldownActive RC->>CRS: CooldownActive()? CRS-->>RC: return true RC->>RC: 返回CooldownActive
(新任务不触发拉起!) RC->>CRS: WaitForCooldownToSettle() Note over RC,CRS: 挂起等待...(条件变量等待) CRS->>CRS: 冷却时间到/被唤醒 CRS-->>RC: RC->>CRS: 状态变为Recovering RC->>CRS: BeginSessionOpen() RC->>ST: OpenSession() ST-->>RC: 返回Ok RC->>CRS: FinalizeSessionOpen() CRS->>CRS: 状态:Active RC->>RC: 继续InvokeAsync RC->>RC: 提交请求到共享内存 RC-->>App: 返回结果

关键说明

冷却期新任务处理特点
  1. 不触发新的拉起：在Cooldown状态下，新任务不会触发新的StartRecovery()
  2. 任务挂起而非失败：调用线程会被阻塞在WaitForCooldownToSettle()
  3. 自动继续：冷却结束后自动进入Recovering，恢复完成后继续处理

冷却实现代码（简化示意）

class ClientRecoveryState {
public:
    // 启动恢复，可能进入Cooldown或Recovering
    void StartRecovery(uint32_t delayMs, uint64_t currentSessionId) {
        const uint64_t cooldownUntilMs = MonotonicNowMs() + delayMs;
        
        // 如果有延迟，进入Cooldown状态
        auto nextState = (delayMs == 0) 
            ? ClientLifecycleState::Recovering 
            : ClientLifecycleState::Cooldown;
        
        // 实际代码：TransitionLifecycle参数顺序和逻辑更复杂
        TransitionLifecycle(nextState, delayMs, currentSessionId, 
                           CooldownWindowChange::Set, cooldownUntilMs);
    }
    
    // 检查是否处于冷却期
    bool CooldownActive() const {
        std::lock_guard<std::mutex> lock(mutex_);
        return MonotonicNowMs() < cooldownUntilMs_;
    }
    
    // 等待冷却结束（简化示意）
    StatusCode WaitForCooldownToSettle(...) {
        std::unique_lock<std::mutex> lock(mutex_);
        
        while (true) {
            if (lifecycleState_ != ClientLifecycleState::Cooldown) {
                return RecoveryStateWaitResultLocked();
            }
            
            if (MonotonicNowMs() >= cooldownUntilMs_) {
                return StatusCode::Ok;  // 冷却结束
            }
            
            // 实际代码：等待条件变量或超时，处理workersRunning等状态
            recoveryStateCv_.wait_until(lock, wakeAt, [...]);
        }
    }
};
